### 实现思路(怎么轻量做)

- **4 张统计卡**:复用顶部统计卡组件
- **全息索引检索**:antd `AutoComplete`,前端模糊匹配 38 个数据集名 + 标签
- **关联分析图**:G6 力导向图,**5~10 个节点**(数据集),边权 hardcode(相关度数字),不需要真做关联挖掘
- **数据集列表**:antd Table,**38 行 hardcode**,字段:名称 / 模态 / 业务标签 / 大小 / 隐私评分 / 关联数 / 操作
- **隐私评分 ε**:数字 + 颜色 tag(ε < 2 → 绿色"安全",2 ≤ ε < 3 → 黄色"中等",ε ≥ 3 → 红色"高风险"),对应 4 层安全里的"数据安全层"
- 「接入新数据集」按钮做成"假接入"(弹窗 + Progress 跑一下,完成后加一条假数据到列表)

### 一句话故事

> "中移动算网平台从各业务系统接入了 38 个数据集。全息索引让多模态数据集毫秒级检索,关联分析告诉运营调度员哪几个数据集联合训练效果最好 —— 比如用户行为日志和交易记录相关度 0.87,适合一起做反欺诈训练。
>
> 每个数据集都带差分隐私评分 ε,这就是 4 层安全评估里的'数据安全层'。"

## 六、跟其他 3 个同学协作的事

### 1. 你是任务流转的起点

整个故事从你这里开始:**运营录入任务 → 调度中枢决策 → 资源池执行 → 边缘协同训练**。

录入任务的动作要往共享 task json 写一条,4 个端口都能看到这条新任务。

### 2. 共用任务 JSON

```json
{
  "task_id": "task-fedtrain-99943",
  "task_name": "联邦训练-图神经网络",
  "type": "联邦训练",
  "status": "pending",
  "business_source": "反欺诈中心",
  "dataset": ["ds-user-behavior", "ds-transaction"],
  "privacy_epsilon": 2.0,
  "noniid_alpha": 0.5,
  "aggregation": "auto"
}
```

### 3. 节点命名层级

你的 1397 个 `node-{city}-{seq}` 节点要挂到任鹏杰的 26 超算中心下。例如 `node-shenzhen-1392` 的父节点是 `supercomputing-shenzhen-01`。表格里加一列"父超算"列展示。

### 4. 数据集命名

数据集命名用 `ds-{业务}-{编号}`,例如 `ds-user-behavior`。
